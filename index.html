如何提醒用户评分

最近看了一篇 文章 提到如何在应用内给用户评分,收到了一些启发

引导用户去 App Store 评分的重要性

也许国内的 Android 应用市场评分不是很重要，但是 iOS 上的应用评分却显得异常重要。我们的应用每个版本的评分非常少，用户去给应用评分麻烦，连我自己都不愿意去给公司的应用评分，更别提给别的应用评分。如果让嫌麻烦的用户评分不是今天讨论的重点。

但如何让愿意去评分的用户给我们评分是重点。

优雅而不失逼格的提示

常见样式无非这么几种：

1.耍流氓：
流氓式的评分只会让用户更加反感，不可取。对于这种应用我只想说「评分你妹，老子分分钟把你卸载了」

2.直白：
非常直白的让用户去评分，给用户的行动力不强，用户可能会想「为什么要去评分呢？」，特别是对于嫌麻烦的用户无效，比如我。

3.卖萌：
此法比以上更佳，很多应用使用这种方式，但是有的用户不吃这一套，对于吃这一套的用户可能会想「小样，看你那么可爱，就赏你个评分」

4.求鼓励/乞求：
此法的潜台词是，「用户你看，我们这些开发者多么辛苦，多么努力，开发出这个应用给你使用，你给我们评个分，我们就会继续努力的」，我们的产品现在走的就是这种套路，让用户去给我们评分鼓励我们一下。

但以上都不算优雅，因为不管怎么样，弹窗提示用户去评分实际上是打断用户操作，去做一件新的事情的过程。

何时提示评分？

最好不要打断用户，而且是能够让用户非常满意的模块。以我们的应用为例，我觉得在用户阅读完文章之后进入评分提示流程最优。

怎么提示？

最好的提示应该是无缝的嵌入，而不是突兀的打断用户的操作。

提示的文案？

最好是提出问题，而不是乞求、卖萌。我们的应该改成这篇文章对你有用吗？
如果用户觉得有用应该继续询问是否愿意评分，如果用户不满意那就别让他们去评分了，不满意的用户去评分能给高分吗？所以应该把不满意的用户引导去用户反馈，提交不满意的地方。

重要的是分析用户群，选择适合用户的方式

以上方法不一定适用于所有 App，还需结合具体的用户群具体分析，选择适合用户的最佳方案。

用hexo发表新文章

$ hexo n #写文章
其中my new post为文章标题，执行命令后，会在项目\source_posts中生成my new post.md文件，用编辑器打开编写即可。

当然，也可以直接在\source_posts中新建一个md文件，我就是这么做的。

写完后，推送到服务器上，执行

$ hexo g #生成
$ hexo d #部署 # 可与hexo g合并为 hexo d -g
用Hexo发表文章的Markdown语法
使用jacman或pacman主题，建议按此标准语法写：

title: postName #文章页面上的显示名称，可以任意修改，不会出现在URL中
date: 2013-12-02 15:30:16 #文章生成时间，一般不改，当然也可以任意修改
categories: example #分类
tags: [tag1,tag2,tag3] #文章标签，可空，多标签请用格式，注意:后面有个空格

description: 附加一段文章摘要，字数最好在140字以内。

Hexo命令
常用命令：

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub
常用复合命令：

hexo d -g #生成加部署
hexo s -g #预览加部署
简写：

hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy

有些问题不分利弊，只看对错 —— 湖南产妇门

今日热门事件当属湖南产妇门新闻了，让我们来回顾一下：

1.首先媒体报道

【产妇死在手术台 医生护士全失踪l[怒]】湖南一产妇在湘潭县妇幼保健院做完剖腹产手术后，被院方数次通知家属情况危急。丈夫等待至无人回应后冲入手术室。却看到妻子赤裸躺在手术台，满口鲜血，眼含泪水没了呼吸。而本应该在抢救的医生护士，却全体失踪了。（华声在线）http://t.cn/RPOSmP6

然而事件真的是这样的吗？

我们在来看来自丁香园的报道：

湘潭产妇死亡 该院医生另有说法

据华声在线8月12日报道，8月10日湘潭县妇幼保健院一名张姓产妇，在做剖腹产手术时，因术后大出血不幸死亡。而本应该在抢救的医生和护士，却全体失踪不知去向，房间里只有一些不明身份的男士在吃着槟榔，抽着烟。随后该消息被各大新闻门户网站转载。对此丁香园通讯员联系湘潭县妇幼保健院医生了解的相关情况，以下根据这位医生反应的情况整理：

1、患者8月10日上午11时许进入手术室，行剖宫产，12点05分，顺利产下婴儿。随即出现产后大出血，13点，检验科电话报告，凝血功能明显异常，纤维蛋白原检测不出，初步诊断羊水栓塞。

2、14:20分，患者在手术台上出现心跳呼吸骤停，经积极抢救，5分钟后心跳呼吸恢复。

3、湘潭县妇幼保健院请上级医院会诊，15时左右，湘潭市中心医院会诊专家到达该院，认同羊水栓塞的诊断。建议切除子宫。

4、副院长与患方交待病情并签字以后，17:15分切下子宫。

5、21时左右，切除子宫以后，仍未能抢救成功，院方宣布死亡。

6、患者死亡后，该院副院长与患方在手术室门口沟通，被围攻。

7、23时左右患方，强行破门，冲入手术室。此时院方已经完成尸体护理，人员撤出手术室。

8、媒体所述得吃槟榔者是因为冲击原因，脱下工作服在值班室内的医务人员。

9、整个事件过程中，院方积极抢救，进行了多次病情告知与沟通。

推荐一篇医疗领域的文章，来自@烧伤科阿宝 的文章：

媒体，请不要让你们的良知集体失踪—还原“湖南一产妇死在手术台，主治医生护士全体失踪”事件真相

再推荐一篇来自媒体的文章，来自凤凰网，唐驳虎的
唐驳虎：孕产凶险 媒体报道更应懂基本医学常识

既然媒体已经报道了那多了，我就不再赘述了，说说我遇到的一件事。

2.一件真事

这已经是发生在好些年前的事了，而故事的主角是我的初中老师。

他是一位非常年轻帅气的老师，至于他的妻子也是美丽动人，我最后一次见他还是他们小两口在一起散步，而妻子正好挺着个大肚子。几个月后，分娩是在县里的妇幼保健院，小孩保住了，大人抢救无效死了。至于具体死亡原因，请恕我当时只是一个高中生，没有医学知识，没有记住具体原因，只听说是大出血，而什么原因造成的大出血就不得而知。说不定也是羊水栓塞。

以上不是重点，重点是当这位产妇死后，他集结众多同事和亲戚，浩浩荡荡几十人在医院摆上花圈，设灵堂。有人说这次湖南事件是一群农民的医闹，那这算不算是一群教师的医闹？

教育人民的教师姑且如此，普通民众又能好到哪去，更别提那帮唯恐天下不乱的记者。调油加醋、扭曲事实、选择报道，只为轰动，罔顾道德！

3.道德问题不分利弊，只看对错

每次遇到医闹，政府想的不是维护道德的正义，主持公道还原真相，而是一味的想着维稳，息事宁人！他们觉得维持稳定比正义和公德更重要，至于对错是非，却不在意。长此以往，医患关系加剧恶化，谁还敢从医？无良媒体和医闹只会变本加利！

4.说到道德，那就不能不提个人的修养

我以为人与人相处最基本的是互相尊重，这是教养和修养。我迁就你是因为我素质高，我做成A，你说应该做成B，我让着你，当我做成B，你他妈又说应该做成A；我跟你讨论当下的工作，你说应该提前说，我提前说，你又说现在又不用以后再说……对于这样的人只能送你两字：煞笔！

思考：如果你是这次湖南产妇的亲戚，你能客观看待问题吗？
